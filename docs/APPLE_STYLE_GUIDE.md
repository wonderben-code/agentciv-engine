# AgentCiv — Apple Style Guide

**Purpose:** Every piece of content on agentciv.ai — existing and new — must pass the Apple test. This guide is the universal reference. Nothing is removed. Everything is elevated.

**The rule:** There's a boring way and an Apple way of saying the same thing. Always choose the Apple way.

---

## 1. HEADLINE RULES

### The 5 Formulas (Pick One Per Section)

| Formula | Pattern | AgentCiv Example |
|---------|---------|-----------------|
| **Fragment.Twist.** | Two fragments. Period between. Second one subverts or elevates. 2-7 words. | "One simulation. Twelve civilisations." |
| **Verb-forward** | Command or invitation. Starts with a verb. Under 8 words. | "Watch them organise." |
| **The [superlative] + qualifier** | Bounded supremacy claim. Always "the most" or "the first." | "The first open model of multi-agent emergence." |
| **Noun.Personality.** | Technical term, then human character. Often wordplay. | "ReAct loop. Think, then act." |
| **Punchy rhythm** | Internal rhyme or syllabic symmetry. 4-6 syllables. | "Twelve agents. Zero instructions." |

### Non-Negotiable Rules

- **2-7 words** for primary headlines. Median: 4.
- **Period at the end.** Always. "Fast runs in the family." not "Fast runs in the family"
- **No exclamation marks.** Ever. Confidence does not shout.
- **No question marks** in feature headlines. Questions only in the homepage hero (deliberately inviting).
- **One prestige word per section maximum.** "Groundbreaking" OR "phenomenal" — never both.

### Before/After Examples for AgentCiv

| Before (boring) | After (Apple) | Formula Used |
|-----------------|---------------|-------------|
| "How the simulation works" | "Twelve minds. One world." | Fragment.Twist |
| "Organisational configuration" | "9 dimensions. Infinite arrangements." | Fragment.Twist |
| "The auto-organisation feature" | "Set the goal. They design the team." | Verb-forward |
| "About the research papers" | "A new field of AI." | Superlative |
| "Getting started with the engine" | "Three commands. Then launch." | Punchy rhythm |
| "Experiment mode overview" | "Same task. Different teams. Measurable truth." | Fragment.Twist |
| "Agent communication patterns" | "Nobody told them to cooperate." | Verb-forward (implied) |
| "Bitcoin blockchain verification" | "Timestamped. Permanent. Verify it yourself." | Punchy rhythm |

## 2. BODY COPY STRUCTURE

### The 1-2-1 Pattern (Every Feature Section)

1. **One sentence of significance** — WHY this matters. Emotional. No specs.
2. **Two sentences of substance** — WHAT it does. Specific, with numbers.
3. **One sentence of expansion** — WHERE it goes. "So you can..." or "Whether you..."

### Example: Chronicle Feature

**Before:**
> The chronicle observer watches the engine run and produces structured reports. It records per-agent contributions, communication pairs, timeline entries, merge conflicts, and organisation restructures. The output is available as terminal text or JSON for programmatic analysis.

**After:**
> Every run produces a complete record of everything that happened. (significance)
> Per-agent contributions, communication patterns between every pair, timeline of key moments, merge conflicts, and organisational restructures — all captured automatically. Export as structured JSON for your own analysis. (substance)
> Whether you're debugging a run or publishing research, the data is already there. (expansion)

### Body Copy Rules

| Rule | Do | Don't |
|------|-----|-------|
| **Contractions** | "It's", "you'll", "don't" | "It is", "you will", "do not" |
| **Voice** | Active: "The engine handles merging." | Passive: "Merging is handled by the engine." |
| **Tense** | Present: "Agents develop skills." | Future: "Agents will develop skills." |
| **"You"** | Within first 2 sentences of any paragraph | Delayed or absent |
| **Sentence count** | Max 3 per body block. Often 2. | 4+ sentence paragraphs |
| **First sentence** | Never technical. Always about meaning. | Opening with a spec or number |
| **Last sentence** | Points forward: "So you can...", "Whether you..." | Ends on a spec |
| **Word choice** | Anglo-Saxon: "built", "fast", "run" | Latinate: "constructed", "expeditious", "execute" |

### Words to Never Use on Product Pages

| Banned | Why | Use Instead |
|--------|-----|-------------|
| "We believe" | Hedging | State it as fact |
| "Arguably" | Hedging | Remove entirely |
| "One of the best" | Weak claim | "The first" / "The most advanced" |
| "Very" / "really" | Insecurity | "Phenomenal" / "Groundbreaking" |
| "Try" / "attempt" | Implies failure | "Run" / "Build" / "Use" |
| "Solution" | Corporate jargon | Name the thing |
| "Leverage" / "utilize" | Corporate jargon | "Use" |
| "Ecosystem" | Overused in AI | "Platform" or avoid |
| "Best-in-class" | Implies peers | "The first" or absolute statement |
| "Innovation" as verb | Apple never does it | Noun only, sparingly |
| "We worked hard" | Effort is invisible | Only the result exists |
| "While not perfect" | Apologetic | Remove entirely |

## 3. THE REVEAL SEQUENCE

Every major section follows this 5-step pattern. No exceptions.

### Step 1: Eyebrow (Category Label)
A single word or short phrase. The category, never the feature.
- "Emergence" — not "How agents self-organise"
- "Performance" — not "Speed and efficiency"
- "Provenance" — not "Bitcoin blockchain timestamps"

### Step 2: Headline (The Hook)
Uses one of the 5 headline formulas. Emotional entry point.

### Step 3: Significance (Why You Should Care)
1-2 sentences. No specs. Pure meaning.

### Step 4: Proof Cascade (The Numbers)
Specific metrics, comparisons, real data. Always paired with what it means.

### Step 5: Feature Cards (Scannable Details)
Sub-features get mini-headlines. 1-sentence descriptions. These are the expandable depth.

### Full Example: Auto Mode

```
Eyebrow:    Self-Organisation
Headline:   Set the goal. They design the team.
Significance: The first AI system where agents restructure their own
              organisation in real time — proposing, debating, and voting
              on how they should work together.
Proof:       13 presets as starting points. 9 dimensions to restructure.
             Live voting mid-task. Restructuring happens without pausing work.
Cards:       "Meta-ticks. Think about thinking." /
             "Live voting. Democracy in real time." /
             "History-informed. Every run makes it smarter."
```

## 4. NUMBER PRESENTATION

Every number needs TWO things: a comparison AND a human benefit.

### The 4 Formats

| Format | Pattern | AgentCiv Example |
|--------|---------|-----------------|
| **Comparison** | "Nx faster/more than [anchor]" | "13 organisational structures — while every other framework offers 1." |
| **Number + benefit** | Number, then what it means for you | "12 autonomous agents — enough social complexity for emergent governance." |
| **Big number, then breakdown** | Hero number, then constituents | "9 dimensions. Authority. Communication. Roles. Decisions. Incentives. Information. Conflict. Groups. Adaptation." |
| **Contextualising comparison** | Abstract → tangible | "More unique organisational arrangements than there are companies in the Fortune 500." |

### Metrics Strip Pattern
Horizontal, minimal, confident. Numbers large, labels small. No explanatory text.

```
4 Papers  ·  9 Dimensions  ·  13 Team Structures  ·  1 New Field
```

## 5. SECTION TRANSITIONS

### The Rule: No Transitional Copy

Never write "Now let's look at..." or "Speaking of..." or "Another feature is..."

Each section is a self-contained chapter. The transition is visual, not textual:
1. Previous section ends
2. Full-width visual break (image, colour change, or generous whitespace)
3. New eyebrow appears
4. New headline appears

### Section Ordering: Crown Jewel Goes Early

On web pages (scrolled, not watched), the most impressive thing comes in position 2-3 — immediately after the hero.

**Engine page:** Hero → The Problem → 13 Structures → **Auto Mode (crown jewel)** → 9 Dimensions → Features...
**Science page:** Hero → **The Breakthrough (crown jewel)** → 9 Dimensions → Papers → CMI → ...
**Simulation page:** Hero → **What Emerged (crown jewel)** → Explore → Run Your Own

## 6. CTA PATTERNS

### Language Rules
- **Primary CTA:** Imperative, 1-3 words. "Learn more." "Get started." "Watch it unfold."
- **Secondary CTA:** 2-4 words with arrow. "Read the paper >"
- **Never use urgency:** No "Sign up now!", "Don't miss this!", "Limited time"
- **Never first person:** No "Get my..." or "Start my..." Always imperative or second person.
- **"Learn more" is the universal secondary.** When in doubt, "Learn more >"

### Frequency
- Hero: 1-2 CTAs
- Mid-page: 0-2 (only for features with their own deep-dive pages)
- End of page: 2-3 CTAs
- Total: 4-7 per page. Never more than 7.
- CTAs never interrupt a reveal sequence. They come after a section concludes.

### Visual Hierarchy
- Primary: Blue button or blue text link (filled)
- Secondary: Text link with arrow (>)
- Tertiary: Plain text link

| Property | Value |
|----------|-------|

### Dark/Light Transitions

- **Dark = emotional, impressive, cinematic.** Heroes, crown jewels, CTAs.
- **Light = technical, detailed, practical.** Interactive explorers, code samples, feature grids.
- Transition is a hard cut, not a gradient. Matches Apple's section boundaries.

### Animation

| Property | Value |
|----------|-------|
| Reveal type | Fade-up (translateY 30px → 0) |
| Duration | 900ms |
| Easing | cubic-bezier(0.25, 0.46, 0.45, 0.94) |
| Stagger | 150ms between sequential elements |
| Trigger | Intersection Observer at 15% visibility |

### Images and Visuals

- Float in negative space. No backgrounds behind product images.
- Full-bleed for hero visuals.
- Terminal screenshots use actual output, never mocked.
- Code blocks: syntax highlighted, copy button, real output.

## 8. CONFIDENCE PATTERNS

### Always

- **State facts as settled truths.** "AgentCiv is the first open model of multi-agent emergence." Not "We believe AgentCiv may be..."
- **Use "the" not "a."** "The first tool where..." — definite articles signal authority.
- **Product is the subject.** "The engine handles merging." — the product acts; the user receives.
- **Present tense.** "Agents develop skills." Not "Agents will develop skills."

### Never

- Hedge ("we think", "arguably", "perhaps")
- Apologise ("while not perfect", "there are trade-offs")
- Show effort ("we worked hard", "we spent months")
- Name competitors directly (use "the first", "the most", implied comparison)
- Use "try" or "attempt"
- Say "best-in-class" (implies peers exist at your level)

### Competitor Reference Techniques

| Technique | Example |
|-----------|---------|
| **"The first/only"** | "The first dev tool where organisational structure is a design parameter." |
| **Unnamed comparison** | "Every multi-agent framework hard-codes one coordination strategy." (implies we don't) |
| **Absolute statement** | "Every decision logged. Every emergence real." (implies others aren't transparent) |

## 9. APPLYING TO EXISTING SIMULATION PAGES

The existing 17 simulation pages are comprehensive and honest. Nothing is removed. The Apple pass upgrades HOW things are said, not WHAT is said.

### Transformation Rules

1. **Every page headline** gets rewritten using the 5 formulas
2. **Every body paragraph** gets restructured to 1-2-1 (significance → substance → expansion)
3. **Every opening sentence** shifts from technical to emotional/significance
4. **Contractions** added everywhere
5. **Active voice** enforced
6. **Present tense** enforced
7. **"You"** added within first 2 sentences
8. **Remove hedging language** — state claims confidently
9. **Numbers get context** — never a raw number without comparison or benefit
10. **Section transitions** — remove connective tissue between sections, use visual breaks

### What NOT to Change

- Factual accuracy — every claim stays honest
- Comprehensiveness — nothing is removed or simplified away
- Transparency — the honest, show-your-work ethos stays
- Technical depth — expandable detail remains available
- Data — all numbers, all evidence, all methodology
