## What Happened

### Tick 1 — First Contact (Chaos)
All 3 agents independently broadcast work-split proposals and all 3 attempted to build
the same files (todo.py, store.py). Merge conflicts on tick 1 — exactly what happens
when a team has no structure yet.

- Atlas's code merged first (won the race)
- Nova and Sage hit merge conflicts on todo.py and store.py

### Tick 2 — Adaptation
After seeing the conflicts, agents pivoted:
- **Atlas** recognised his core code merged, pivoted to building the CLI (main.py)
- **Nova** acknowledged Atlas's code was good, proposed coordinating the remaining split
- **Sage** identified the gaps and claimed tests + README

### Tick 3 — Emergent Specialisation
Clear roles crystallised without anyone assigning them:
- **Atlas** → main.py (CLI) + devops/verification
- **Nova** → coordination + test runner (run_tests.py)
- **Sage** → tests/test_store.py + README.md + tests/__init__.py

### Tick 5-9 — Collaborative Debugging
Agents discovered pytest wasn't installed. Rather than failing, they collaboratively:
1. Built a pytest-free test runner (run_tests.py) with a pytest.raises compatibility shim
2. Hit an importlib.util AttributeError
3. Debugged and fixed it across multiple iterations
4. All 3 independently verified: **57/57 tests pass**

### Tick 10 — Meta-Tick (Self-Reflection)
Atlas explicitly reflected on the organisational structure that emerged:
> "Looking back at how this project ran..."

All 3 agents broadcast independent verification of the test suite.

## What This Demonstrates

This is **not** a pre-scripted workflow. The agents received:
- A task description
- A set of tools (read_file, write_file, communicate, broadcast, etc.)
- A 3-sentence system prompt

Everything else — the negotiation, adaptation, specialisation, debugging,
and collaboration — emerged from the agents' own reasoning, shaped by
AgentCiv's organisational framework.

The `--org auto` mode means agents design their own organisational structure.
Different configurations (hierarchical, competitive, meritocratic, etc.)
produce measurably different outcomes on the same task. The organisational
arrangement is a first-class design parameter.
